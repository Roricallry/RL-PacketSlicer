>强化学习（Reinforcement Learning, RL）是让一个智能体（Agent）通过反复试错，从环境中获得奖励或惩罚，自己学会“什么状态下该做什么动作”的方法。

##### 四个关键要素（每个项目都必须定义清楚）

| 要素      | 含义                 | 举例（比如你做一个“自动玩贪吃蛇”的项目）      |
| ------- | ------------------ | -------------------------- |
| **智能体** | 你要控制的那个决策者         | 贪吃蛇的“大脑”                   |
| **环境**  | 智能体所在的世界，它会反馈状态和奖励 | 游戏界面、墙壁、食物、蛇身              |
| **状态**  | 某一时刻环境的关键信息        | 蛇头坐标、食物位置、前进方向、是否撞墙        |
| **动作**  | 智能体每步能做的事          | 上、下、左、右移动                  |
| **奖励**  | 环境给智能体的分数（正/负）     | 吃食物 +10；撞墙 -100；每存活一步 +0.1 |
##### 强化学习的运行流程（循环）
1. 智能体观察当前 **状态**（S）
2. 根据某种 **策略**（Policy，相当于它的“行动指南”）选择一个 **动作**（A）
3. 动作作用于环境，环境更新成新状态（S’），并返回一个 **奖励**（R）
4. 智能体收到奖励，调整自己的策略，让以后更可能选择“高奖励”的动作
5. 重复1～4步，直到学会最优策略

##### 核心思想
###### 1. 延迟奖励（Credit Assignment）
有些动作的“好”要在很多步之后才体现。例如下棋：当前这一步走车，可能10步后才导致胜利。RL需要能够把未来的胜利“归功于”前面的动作。
###### 2. 探索 vs. 利用（Exploration vs. Exploitation）
- **利用**：用已经知道的好方法（走当前最优路线）
- **探索**：尝试没做过的新动作，也许能找到更好的路线
项目初期要设置一定的探索率（比如10%随机动作），防止智能体错过最优解。
###### 3. 马尔可夫决策过程（MDP）
听起来高级，其实就是：“未来只取决于**当前**状态和当前动作，和更早的历史无关。”大部分RL问题都这样设定。


#### 本项目映射

|要素|定义|本项目中的具体实现|
|---|---|---|
|**环境 (Environment)**|智能体与之交互的外部系统|整个网络：背景负载发生器、分割器、IoT 设备流量、重放探针等|
|**状态 (State, `s`)**|智能体在时刻 `t` 观察到的环境特征|5维向量：`[U, R, L_base, L_base_max, last_p]`  <br>- `U`：全局最大利用率 (0~100%)  <br>- `R`：IoT 流量归一化熵 (0~1)  <br>- `L_base`：ICMP 平均延迟 (ms)  <br>- `L_base_max`：ICMP 最大延迟 (ms)  <br>- `last_p`：上一轮选择的分割概率|
|**动作 (Action, `a`)**|智能体可执行的决策|选择 `p_value`，即分割器触发分割的概率，取值范围 0.00~1.00（离散化为 101 个动作）|
|**奖励 (Reward, `r`)**|环境反馈给智能体的分数，评价动作好坏|`r = w * R_norm - (1-w)*(2 * L_norm + p_max)`  <br>- 高熵 → 正奖励  <br>- 高延迟/抖尾 → 负惩罚|
|**策略 (Policy, `π`)**|智能体的决策规则，从状态到动作的映射|一个神经网络（Actor），输入状态，输出每个动作的概率分布|
|**经验 (Experience)**|单个时间步的交互记录|`(s, a, r, s')`，由控制器采集后发送给 PPO 服务器|
|**价值 (Value, `V(s)`)**|预期从状态 `s` 开始能获得的累计奖励|另一个神经网络（Critic），用于估计状态的好坏|
